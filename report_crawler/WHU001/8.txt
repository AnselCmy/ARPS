环节一    演讲：Programming Should Be More Than Coding
Leslie Lamport博士， 2013年”图灵奖”得主（图灵奖被誉为计算机领域的诺贝尔奖）、美国国家科学院和国家工程院院士、LaTeX系统创始人。他是拥有杰出贡献和辉煌成就的计算机大师，他关于时间时钟、面包店算法、拜占庭将军等问题的思考令人乍舌，他的分布式计算理论更是奠定了计算机学科的基础。他于1978年发表的《分布式系统内的时间、时钟和事件顺序》至今保持着计算机学科史上的论文被引用量纪录。
比尔∙ 盖茨对Lamport博士的贡献也做出了很高的评价:“ Leslie的伟大不仅在于计算机科学领域，而且还体现在他让世界变得更加安全的努力中，他的工作正在让世界各地无数的人们受益。” Lamport博士目前是微软研究院的一名首席研究员。
演讲摘要: Writing a program involves three tasks: 1. Deciding what the program should do. 2. Deciding how the program should do it. 3. Coding: Implementing these decisions in code. Too often, all three are combined into the process of coding. This talk explains why they should be separated, and discusses how to perform the first two.
